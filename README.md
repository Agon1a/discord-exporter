# 💬 Discord Exporter

[![Tests](https://github.com/Agon1a/discord-exporter/actions/workflows/tests.yml/badge.svg)](https://github.com/Agon1a/discord-exporter/actions/workflows/tests.yml)

🌍 [English version](README.en.md)

Discord Exporter - браузерное расширение для Chrome и Edge на Manifest V3. Расширение экспортирует сообщения из Discord Web за выбранный диапазон дат в CSV, JSON или Markdown.

Экспорт выполняется локально в активной вкладке браузера. Данные сообщений не отправляются на сторонние серверы.

## ✨ Возможности

- 📅 Экспорт сообщений за выбранный диапазон дат.
- 📦 Форматы вывода: CSV, JSON, Markdown.
- 🔄 Обычный режим с прокруткой истории канала.
- ⚡ Быстрый режим через текущую сессию Discord, если доступен API Discord Web.
- ↩️ Автоматический возврат к обычному режиму при недоступности быстрого режима.
- 🔎 Фильтрация по автору и ключевым словам.
- 🧩 Режим AND/OR для ключевых слов.
- 🤖 Исключение сообщений ботов.
- 🧹 Удаление дубликатов по дате, автору и тексту.
- 🧭 Определение открытого сервера и канала.
- 📊 Индикатор прогресса экспорта.
- 🌓 Светлая и темная тема с сохранением выбора в `localStorage`.
- ⏱️ Быстрый выбор дат: сегодня, 7 дней, 30 дней, год.
- 🌐 Поддерживаемые домены: `discord.com`, `ptb.discord.com`, `canary.discord.com`.

## 🚀 Установка

1. Перейти на `chrome://extensions` или `edge://extensions`.
2. Включить режим разработчика.
3. Нажать **Load unpacked**.
4. Выбрать папку репозитория `discord-exporter`.
5. После загрузки расширения открыть Discord Web и закрепить значок расширения при необходимости.

## 📖 Использование

1. Открыть Discord Web и перейти в нужный канал.
2. Открыть popup расширения.
3. Задать даты **С** и **По**.
4. Выбрать формат экспорта: CSV, JSON или Markdown.
5. При необходимости открыть **Расширенные фильтры** и настроить условия отбора.
6. Для экспорта через прокрутку оставить быстрый режим выключенным.
7. Для экспорта через API включить быстрый режим.
8. Нажать **Экспортировать**.
9. После завершения будет скачан файл с экспортированными сообщениями.

## ⚡ Режимы экспорта

| Режим | Поведение | Ограничения |
|---|---|---|
| Обычный | Страница прокручивается вверх, сообщения собираются из DOM | Скорость зависит от объема истории, сети и производительности браузера |
| Быстрый | Сообщения запрашиваются через текущую сессию Discord Web | Может быть недоступен; личные сообщения не поддерживаются |

Если быстрый режим недоступен, расширение переключается на обычный экспорт.

## 📤 Форматы

| Формат | Назначение |
|---|---|
| CSV | Табличная обработка в Excel, Google Sheets и аналогичных инструментах |
| JSON | Структурированные данные с метаданными экспорта |
| Markdown | Читаемый текстовый архив сообщений |

### 📊 CSV

```csv
"Дата","Автор","Текст"
"2024-01-15T10:30:45.000Z","Alice","Hello!"
"2024-01-15T10:31:12.000Z","Bob","Hi there!"
```

### 🧾 JSON

```json
{
  "exportDate": "2024-01-15T10:30:45.000Z",
  "format": "json",
  "metadata": {
    "channelName": "general",
    "serverName": "My Server",
    "messageCount": 2,
    "dateRange": {
      "from": "2024-01-01",
      "to": "2024-01-31"
    }
  },
  "messages": [
    {
      "timestamp": "2024-01-15T10:30:45Z",
      "author": "Alice",
      "content": "Hello!"
    }
  ]
}
```

### 📝 Markdown

```markdown
# My Server - general

> **Exported:** Jan 15, 2024
> **Date Range:** 2024-01-01 to 2024-01-31
> **Messages:** 2

#### Alice - 2024-01-15T10:30:45.000Z
Hello!
```

## 🔍 Фильтры

Фильтры находятся в блоке **Расширенные фильтры** popup-интерфейса.

| Фильтр | Описание |
|---|---|
| Автор | Поиск по имени автора; поддерживается строка или регулярное выражение |
| Ключевые слова | Поиск фраз в тексте сообщения через запятую |
| AND для ключевых слов | Требуются все указанные слова |
| OR для ключевых слов | Достаточно одного указанного слова |
| Исключить сообщения ботов | Сообщения ботов исключаются из результата |
| Удалить дубликаты | Повторяющиеся сообщения удаляются по ключу дата + автор + текст |

## 🔒 Приватность

- Обработка выполняется в браузере.
- История сообщений не передается на серверы проекта.
- Для быстрого режима используется активная сессия Discord Web в текущей вкладке.
- Расширение работает только на доменах Discord, указанных в `manifest.json`.

## ⚠️ Ограничения

- Экспорт зависит от структуры DOM Discord Web.
- При изменении разметки Discord могут потребоваться правки селекторов.
- Длинная история экспортируется медленнее из-за пошаговой прокрутки.
- Быстрый режим доступен не во всех каналах и не поддерживает DM.
- Firefox не указан в `manifest.json` как целевая платформа.

## 🛠️ Траблшутинг

| Проблема | Решение |
|---|---|
| `Receiving end does not exist` | Обновить страницу Discord и повторить экспорт |
| Сообщения не найдены | Проверить диапазон дат и дождаться полной загрузки канала |
| CSV не скачивается | Проверить настройки загрузок браузера |
| Прогресс не меняется | Подождать несколько итераций прокрутки; при зависании обновить страницу |
| Быстрый режим не работает | Использовать обычный режим |

## 📁 Структура проекта

- `manifest.json` - конфигурация расширения, разрешения, content scripts, popup.
- `public/popup.html` - разметка popup-интерфейса.
- `public/styles.css` - стили popup-интерфейса.
- `public/icons/` - иконки расширения.
- `src/popup.js` - запуск popup-логики.
- `src/content.js` - запуск content script.
- `src/background.js` - фоновые обработчики расширения.
- `src/lib/popup-utils.js` - логика popup, быстрый режим, скачивание файла.
- `src/lib/content-utils.js` - прокрутка, сбор сообщений, прогресс.
- `src/lib/shared-utils.js` - общие утилиты для дат, CSV и уникальных ключей.
- `src/lib/selectors.js` - централизованные селекторы Discord DOM.
- `src/lib/error-handler.js` - обработка и нормализация ошибок.
- `src/lib/filters.js` - фильтрация сообщений.
- `src/lib/i18n.js` - локализация интерфейса.
- `src/lib/performance.js` - вспомогательные функции производительности.
- `src/lib/templates.js` - шаблоны экспорта.
- `src/lib/batch.js` - модель batch-экспорта.
- `src/lib/history.js` - модель истории экспортов.
- `src/lib/exporters/` - экспортеры CSV, JSON и Markdown.
- `tests/` - unit-тесты.
- `scripts/release.js` - скрипт подготовки релиза.

## 🌐 Локализация

Основные переводы интерфейса доступны для русского и английского языков. Для немецкого, французского и испанского добавлен базовый набор строк. Коды `ja` и `zh` присутствуют в списке поддерживаемых языков, но без отдельной таблицы переводов используются fallback-строки.
